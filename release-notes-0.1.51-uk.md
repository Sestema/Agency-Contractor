# Agency Contractor 0.1.51

У цьому релізі завершено фінальний cleanup фінансового шару: `finance_data.json` більше не використовується як runtime storage, а активна фінансова логіка програми працює тільки через SQLite.

- повністю прибрано runtime-залежність від `finance_data.json` у фінансовому модулі: читання, запис, backup/restore і внутрішній JSON-контейнер більше не використовуються для поточної роботи фінансів;
- `Advances`, `CustomFields`, `Reports`, `Accommodations` і витрати по фірмах тепер працюють як SQLite-only шляхи без legacy fallback на старий фінансовий JSON;
- спрощено `FinanceService` та пов’язані сервіси: видалено застарілу JSON-обв’язку, runtime `_db`-стан і зайві compatibility-шари, що зменшило складність коду та ризик прихованих розбіжностей між storage-paths;
- збережено окрему міграцію старих `salary_*.json` для зарплатних місяців, але сам `finance_data.json` більше не є частиною активного фінансового runtime-потоку;
- підтверджено стабільний запуск і роботу основних фінансових сценаріїв після cleanup: застосунок стартує чисто, а salary/report екрани продовжують працювати через SQLite без ознак повернення до старого storage;
- додатково збережено захист місячних salary SQLite-файлів у сценаріях OneDrive/shared folder, щоб уникати створення дубльованих month DB і ризикового перезапису витрат.
