# Agency Contractor 0.1.53

У цьому релізі зосередилися на фінальній стабілізації фінансового модуля перед публікацією: виправили кілька точкових сценаріїв збереження, посилили діагностику проблемних даних і зменшили ризик тихих помилок у salary-потоці.

- виправлено сценарій, у якому збереження зарплатного місяця могло виглядати успішним навіть без доступного `SalaryDbService`: тепер такий стан коректно вважається помилкою, а не тихим успіхом;
- виправлено порівняння назв фірм при збереженні витрат: фільтрація тепер працює без залежності від регістру, тому `Firm A` і `firm a` більше не розходяться як різні фірми;
- додано warning-логування для невдалого `ParseDecimal()` у локальних SQLite-сервісах, щоб швидше знаходити пошкоджені або нестандартні числові значення без прихованої втрати діагностики;
- збережено безпечну fallback-поведінку для проблемних числових значень: застосунок, як і раніше, не падає, але тепер залишає зрозумілий слід у логах;
- розширено regression-тести для `FinanceMonthPaymentsService`, `SalaryDbService`, `LocalDbService` і view-model guard-логіки, щоб окремо покрити нові сценарії стабільності фінансового потоку.
