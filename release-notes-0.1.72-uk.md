# Agency Contractor 0.1.72 — стабільний PostgreSQL multi-PC і щоденні backup-и

## PostgreSQL multi-PC
- **PostgreSQL multi-PC переведено у стабільний канал** — програма може працювати з однією спільною PostgreSQL базою для кількох ПК.
- **SQLite режим залишився доступним** — стара база працює як раніше для одного ПК або як резервний режим.
- **Покращено перемикання режимів бази** — після зміни SQLite/PostgreSQL програма показує зрозумілу підказку і кнопку перезапуску.

## Стабільність синхронізації
- **PostgreSQL push `LISTEN/NOTIFY` став стійкішим** — listener автоматично перепідключається після тимчасового обриву мережі, VPN, сну ПК або рестарту PostgreSQL.
- **Додано ретраї для PostgreSQL publish-подій** — короткі мережеві збої менше впливають на повідомлення між ПК.
- **Додано TCP keepalive для PostgreSQL-з'єднань синхронізації**, щоб швидше виявляти мертві підключення.

## Backup
- **Додано автоматичний щоденний SQLite backup** у кореневу папку `Backup\YYYY-MM-DD`.
- **Backup створюється тільки один раз на день** — якщо працює 3-5 ПК, спільний lock у папці `Backup` не дозволяє кільком комп'ютерам робити backup одночасно.
- **Зберігаються тільки останні 7 днів** — старі backup-папки автоматично очищаються.
- **У PostgreSQL режимі перед backup програма оновлює SQLite файли з PostgreSQL**, щоб у папці backup була актуальна аварійна копія для переносу або повернення в старий режим.
- **Backup не створюється, якщо PostgreSQL → SQLite експорт не вдався**, щоб не зберегти старі або неповні дані як актуальні.

## Зарплати
- **Виправлено дублювання виплат у картці працівника** — старі дублікати історії зарплат більше не показуються як дві однакові виплати.
- **Нові дублікати `salary_history` не мають створюватися** при збереженні, навіть якщо раніше запис мав інший `employee_id` або зайві пробіли в назві фірми.
- **Покращено роботу з одним місяцем і різними фірмами** — користувачі можуть безпечніше працювати з різними фірмами в одному зарплатному місяці.

## Локалізація і налаштування
- **Долокалізовано нові серверні налаштування і PostgreSQL-повідомлення** для української, англійської, чеської та російської мов.
- **Покращено відображення активної бази** — у налаштуваннях видно, чи поточний запуск реально працює через SQLite або PostgreSQL.
