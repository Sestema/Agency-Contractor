# Agency Contractor 0.1.75 beta — захист зарплат при роботі з кількох ПК

## Зарплати і multi-PC
- **Звичайне збереження зарплат тепер записує тільки змінених працівників**, а не весь місяць.
- **SQLite і PostgreSQL використовують row-level upsert** для зарплатних рядків, щоб старий кеш одного ПК не перезаписував години іншого ПК.
- **Додано захист від старого кешу**: якщо рядок зарплати вже змінено на іншому ПК, програма зупиняє перезапис і просить оновити рядок.
- **Sync-події зарплат стали точнішими**: для невеликих змін передається конкретний працівник, фірма і місяць; повне перезавантаження місяця лишається fallback.

## Стабільність
- **Виправлено зависання loading overlay** при перемиканні місяців після beta `0.1.74`.
- **Зменшено ризик втрати останніх змін** при одночасній роботі кількох ПК у зарплатній таблиці.
- **Папка шаблонів продовжує використовувати вже наявну локалізовану назву**, щоб шаблони не зникали між ПК з різними мовами інтерфейсу.

## Multi-user фундамент
- **Підготовлено перший фундамент для майбутніх користувачів і ролей**: feature flags, tenant/user/role/audit архітектура, optional поля профілю та розширений audit metadata.
- У старому режимі ці multi-user функції вимкнені й не змінюють поведінку програми.
