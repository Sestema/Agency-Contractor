# Agency Contractor 0.1.50

У цьому релізі зосередилися на стабільності фінансового модуля, завершенні активного переходу з `finance_data.json` на SQLite та захисті місячних salary-баз від конфліктів під час роботи через OneDrive.

- завершено перехід активних фінансових сценаріїв на SQLite: `CustomFields`, `Reports`, `Accommodations`, `Advances`, `SalaryHistory` і витрати по фірмах тепер працюють через SQLite-first шляхи;
- прибрано більшість production write-path у `finance_data.json`, тому JSON більше не є основним джерелом збереження для поточної фінансової роботи;
- посилено міграції фінансових даних на старті програми: додано перевірки завершення, журналювання і безпечний fallback, якщо старі дані вже були порожні або перенесені раніше;
- виправлено ризиковий сценарій з місячними salary SQLite файлами в OneDrive: якщо для місяця вже існує один реальний файл на кшталт `salary_2026_03.WORK.db`, програма більше не повинна створювати другий “канонічний” файл поруч;
- додано захист від конфлікту, коли для одного місяця знайдено кілька salary DB файлів: у такому випадку збереження блокується з попередженням, щоб не перезаписати витрати або зарплатні дані не в той файл;
- покращено читання, перерахунок і підготовку salary/report даних, а також стабілізовано кілька гарячих шляхів у фінансових сервісах після великого storage cleanup;
- продовжено UI-polish та стабілізацію ключових екранів: оновлено частину тем, ресурсів, мовних рядків і пов’язаних view/viewmodel шляхів, щоб фінансові та службові екрани працювали надійніше;
- додатково посилено службові частини навколо employee indexing, admin mirror, нормалізації даних та пов’язаних допоміжних сервісів.
